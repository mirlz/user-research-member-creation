Improving player creation user flow process

Brief: To design a better system or workflow where users can:

  • Login quicker
  • Browse / navigate easier regardless of type of devices used
  • Create player account(s) quicker
  • Allow for easier passing of player account(s) details to players

Who are our users?

Brief: Our users are the agents, including MA, SMA, or VVIPs possibly who uses our agent site for:-

  • Manage downline (members / players) accounts' credit / balance / commission
  • Transfer downline balance
  • Checking reports (winloss / statement etc)
  • Member account(s) creation

Heuristic Evaluation

1. Login Flow

Desktop View
Tablet View
Evalation

1.1 Fill in username/ password to login

There is no help for lost username / password.

There is no way for user to see what has been entered into password field already, no way to discover if there are any typo errors.

Heuristic violated:
  • Error prevention
  • Help users recognize, diagnose and recover from errors
  • Match between the website and the real world

1.2 User have to input security code / OTP if current IP does not match last login IP

If last login IP does not match current IP, user will be brought to this page to input either user's preset security code or a unique OTP will be sent to device's preferred contact method.

This is a disruptive step especially if user set a different OTP method as then user would have to either check sms / email to input security code.

Heuristic violated:
  • Recognition or recall
  • Flexibility and efficiency of use

1.3 User have to agree to T&C everytime upon login

When the user successfully login, he will be brought to the "Terms and Conditions" page where he will have to click on "I Agree" button, doing otherwise would log him out and bring him back to the login screen.

This is unnecessary as it's an additional page.

Heuristic violated:
  • Error prevention
  • Aesthetics and minimalist design
  • Flexibility and efficiency of use

2. Member Creation Flow

Desktop View
Tablet View
Evalation

2.1 User directed to landing page after login

On reaching landing page, need to look out for where to navigate to go to create an account.

Heuristic violated:
  • Aesthetics and minimalist design
  • Flexibility and efficiency of use

2.2 User directed to security code / OTP page

If user were not prompted security code after login (due to difference in IP from last login), then on visiting pages under Member Mgmt / Transfer / Account / Security, user would be prompted with security code / OTP verification.

Heuristic violated:
  • Recognition or recall
  • Flexibility and efficiency of use

2.3 Clicking on Member Mgmt > New Member

  1. Need prior knowledge of which account to copy settings from
  2. Have to try multiple times for a unique ID that has not been used
  3. Have to scroll to bottom to check password requirements, no real time check
  4. Optional but not stated, what is required / optional? Do we really need this section?
  5. Outdated recaptcha, hard to read
  6. On error submission, no highlighting for errors. With this many fields, how to enable to bulk creation?
  7. Unorthodox requirement of password not allowed to contain first / last name and username
  • Recognition or recall
  • Aesthetics and minimalist design
  • Match between the website and the real world
    • Help users recognize, diagnose and recover from errors

2.4 Form submission - error

On submission, fields that are empty but required, or does not fulfill the min / max requirements will return an alert error. However, error checking are done on top - bottom / priority basis, as the password requirements are not fulfilled,the error only showed password but did not contain error for credit / max bet that went over the limit.

Heuristic violated:
  • Help users recognize, diagnose and recover from errors
  • Match between the website and the real world

2.5 Form submission - success

Upon successful submmission of form, a member account is created. A successful alert will pop up that indicates that a new account has been successfully created.

On clicking "Ok", the create member page will refresh with all fields reset.

Heuristic violated:
  • Match between the website and the real world
